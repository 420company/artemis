why

Artemis exists to push against recurring problems in coding-agent loops:

  • acting too early after only partial file reads
  • retrying the same failure pattern until tokens disappear
  • burying routing, context, and verification state
  • making new users solve tooling friction before they can do useful work

The goal is a cleaner runtime, clearer workflows, and a friendlier path.

core workflows
  • run — direct one-shot work
  • niko — multi-agent planning, shaping, synthesis
  • athena — broader multi-agent execution
  • contest — adversarial proposal, critique, verdict
  • mind — switch brain/execution layer mid-flow
  • design — frontend direction, UI critique
  • doublekill — dual-model setup
built-in capabilities
  • version-aware docs lookup with docs
  • selectable fallback backend with search-engine
  • Gemini Deep Research as independent research engine
  • Telegram as optional remote control bridge
  • Artemis.MD as repo-level operating memory
  • WordUP autosave snapshots
  • interactive HUD with model, token, context progress
  • verification planning, evidence tracking
  • permission modes + --whosyourdaddy for accept-all
