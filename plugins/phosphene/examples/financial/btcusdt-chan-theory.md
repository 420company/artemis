# Financial Example — BTC/USDT 缠论分析

**场景：** 用户粘贴了 BTC/USDT 的价格描述，请求结构性分析。

**激活：** Phosphene 在检测到金融内容时自动切换 research 预设 + financial-lexicon 模块。

---

## 用户输入

```
BTC/USDT 日线。
5月初从68000开始下跌，最低打到59800，然后反弹到66500，
之后再次下跌到60200，现在反弹到64000。
整体感觉是震荡，但不确定是否已经形成了底背驰。
```

---

## Phosphene 分析输出（research + financial-lexicon）

**Pattern-Reader（结构识别）：**

```
走势还原：
  a: 68000 → 59800  (下跌段，-8200，约 -12.1%)
  b: 59800 → 66500  (反弹段，+6700，约 +11.2%)
  c: 66500 → 60200  (下跌段，-6300，约 -9.5%)
  d: 60200 → 64000  (当前反弹，+3800，约 +6.3%，进行中)
```

按缠论定义，a 和 c 构成**下跌中枢**（重叠区间：60200–66500）。这是标准的震荡结构，尚未判断方向。

**Cartographer（边界绘制）：**

关键价位图：

```
68000 ──────── 前高（结构顶）
66500 ──────── 中枢上沿 / b 段高点
64000 ──────── 当前位置（d 段进行中）
60200 ──────── 中枢下沿 / c 段低点
59800 ──────── a 段低点（结构底候选）
```

如果 d 段突破 66500，中枢向上扩张，趋势可能转多。
如果 d 段终止后再次下跌跌破 59800，中枢向下突破，看更大级别下跌。

**Skeptic（背驰判断）：**

用户问"是否已经形成底背驰"——先定义清楚：

**背驰的必要条件（缠论标准）：**
1. 必须在同级别的两段下跌之间（a 和 c 是同级别 ✓）
2. c 段的 MACD 面积必须小于 a 段（**无数据，无法判断** ⚠️）
3. c 段的低点可以高于或低于 a 段低点均可（c=60200 > a=59800 ✓，低点抬高，加分）
4. 背驰不等于反转，只是该级别下跌段的结束信号

**结论（基于已有信息）：**

- 低点抬高（59800→60200）是**弱背驰信号**，方向支持
- MACD 面积未知是**最大的不确定项**，不能下结论
- 当前 d 段（60200→64000）的形态和成交量未知，无法判断 d 是否是独立走势还是 c 段反弹

**Archivist（历史对照）：**

BTC 在 2023 年 8-10 月有一个结构相近的震荡：29000→24800→28700→25100，最终向上突破。2022 年 6-9 月有一个更深的同结构最终向下破位。同一拓扑，两个结果——结构只是可能性地图，不是预言。

**Semiotics（语言信号）：**

用户用了"感觉是震荡"和"不确定"——这两个词说明用户已经有正确的感知，只是需要结构语言来确认。在这里，磷烯的作用是把直觉翻译成可操作的判断框架，不是提供用户没有的信息。

## 建议的下一步

```
1. 查日线 MACD，对比 a 段和 c 段的柱面积
2. 看 c 段成交量是否缩量（缩量 + 低点抬高 = 更强的背驰信号）
3. 等待 d 段走完再判断——背驰确认需要在 d 段出现明确的顶部结构之后
4. 设置关键观察位：60000（中枢下沿附近）破位则放弃多头假设
```
