# Design System Catalog Pattern

A reusable design-system source should include these sections:

1. Visual theme and atmosphere.
2. Color palette and semantic roles.
3. Typography rules and fallback fonts.
4. Component styling for buttons, inputs, cards, navigation, tags, overlays, and data views.
5. Layout principles, grid, spacing, and density.
6. Depth, elevation, borders, and surface treatment.
7. Do and do-not rules.
8. Responsive behavior and breakpoints.
9. Agent prompt guide for future generation.

## Inspiration Categories

- AI and developer platforms.
- Dashboards and SaaS tools.
- Editorial and media systems.
- Fintech and crypto products.
- E-commerce and retail.
- Automotive and premium product pages.
- Creative tools and portfolio systems.

Use categories only as structural inspiration. Generate original systems unless
the user explicitly asks to analyze a specific source.

## Style Recipe Fields

When adding or selecting an aesthetic, define it as an executable recipe rather
than a name:

- Aliases and trigger phrases.
- Composition cues.
- Typography cues.
- Material and surface cues.
- Color palette and emotional role.
- Motion cues.
- Micro-detail cues.
- Avoid rules.
- Best-fit artifact types.

Use no more than two recipes per prompt by default. If two are used, write the
dominant/subordinate ratio so layout, color, and material rules do not fight.
